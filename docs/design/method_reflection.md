# SchemaMem 方法反思:12 轮迭代,收敛到优雅形式化

> 目标(用户指令):方法优雅简洁(数据结构、算法数学上严谨且好讲述)、有效、服务于 SchemaMem 立意。
> 方式:对当前 `core.py`(候选分桶 + k=2 计数 + finalize 扫描 PROTECT)逐轮找真实弱点并改。
> 立意红线(锁定):单一预测残差判据统管;不是三招堆叠;例外是"仲裁在发生"的存在性证明;残差=图式期望被违反的程度。

## 收敛结论先行(读这一段就够)

**一个槽位 = 一列带残差的断言;一个信号 = 残差 r;两个轴 = 残差幅度 × 跨 episode 复现。** 沿这两个轴,整个动作集自然落位,不需要五条独立规则:

| | r ≈ 0(可预测) | r 高(违反) |
|---|---|---|
| **复现** | ASSIMILATE(确认信念)/ DISSOLVE(冗余可丢) | ACCOMMODATE(图式错了,改信念) |
| **孤立** | 同上 | PROTECT(真但不可归纳,留作例外) |
| 过渡态 | — | ACCUMULATE(违反已现、复现性未判定) |

r 低 → 确认+可压缩(同化/遗忘);r 高 → 按"是否复现"劈成 顺应 vs 例外。**这就是全部。** 一个信号 + 一个修饰(复现性),四个内容动作 + 遗忘全部导出。这是本文档 12 轮迭代的落点,也是最该写进方法章开头的一张表。

两条支撑它的严谨性支柱(替换掉"k=2 是我拍的"):
- **可识别性下界**:单条观测在数学上无法区分"永久改变"与"一次例外";要估计"持久性"至少需 2 个独立 episode。故"从不首次即改"不是保守,是**信息论上必需**。
- **MDL 阈值**:达到可识别性下界后,当"改写信念省下的编码 > 切换成本"时才顺应。k 不再是魔数,而是 ⌈切换成本 / 单次残差⌉。

## 迭代 1|数据结构:belief 与 candidate 是二元冗余 → 统一为"假设"

**弱点(真实,在 `core.py`):** belief 是一个裸字符串,candidate 是对象;为了让顺应后同线观测能同化,加了 `won_lines` 集合当补丁。三个字段(`belief` / `candidates` / `won_lines`)描述的是同一件事——"这个槽位见过哪些值、各有多少支持"。补丁是不优雅的信号。

**改法:** 槽位只存**一列断言** `assertions = [(value, episode, r, t)]`,派生出**假设** `H_v = {断言过 v 的 episode 集合}`。于是:
- `belief` = 当前被指派为"在任"的假设(一个指针,不是独立字符串);
- `superseded` = 历任在任假设的序列;
- `exceptions` = 支持数停在 1、从未在任的假设;
- `candidates` / `won_lines` **消失**——它们都是这列断言上的**视图**。

belief 不可能再和证据 desync(它就是证据的 argmax 视图)。`won_lines` 补丁自然蒸发:同线观测只是给在任假设的 episode 集合 +1 = 同化。

## 迭代 2|算法:k=2 是魔数 → 可识别性下界 + MDL 阈值

**弱点:** `k=2` 是硬编码整数,审稿人必问"为什么是 2"。当前 docstring 甚至说不出所以然。

**改法(两段论,都严谨):**
1. **可识别性:** 给定某槽位当前值 A,来一条 B。B 是"A→B 的永久改变"还是"A 上的一次例外"?**一条观测下,这两个假设产生完全相同的似然**——不可识别。要估计"B 是否持久",必须有第二个独立 episode 再次断言 B。所以 **k ≥ 2 是硬下界,与参数调节无关**。这把"推迟提交"从工程保守升级成数学必需。
2. **MDL:** 到达下界后,是否继续攒还是立即顺应,由描述长度决定。保留旧信念、把每条违反当残差显式编码,成本 ≈ n·r̄(n 条违反,均残差 r̄);切换信念成本 = S(编码新信念 + supersede 链)。当 n·r̄ > S 即切换,得 **k\* = ⌈S / r̄⌉**。含义:违反越强(r̄ 大),需要的复现越少——但**永不小于可识别性下界 2**。

**副产品(可测的理论预言):** 强违反使 k\* 逼近 2,弱违反使 k\* 变大。这是一个能在探针上验证的定量预言,不是事后叙事。

## 迭代 3|框架-实现错配:BOCPD 讲得太大,实现只是计数器

**弱点:** 之前把 BOCPD(贝叶斯在线变点、run-length 后验)当理论招牌,但实现是"数独立 episode ≥ k"。这个落差审稿人会抓。

**改法(诚实分层):** 主张的机制就是**累积证据的序贯检验**(sequential test):对每个竞争假设累积其跨 episode 支持,越过 MDL+可识别性阈值则切换。CUSUM / Page-Hinkley / BOCPD 是它在**连续残差流**上的推广,正文写作"我们的计数规则是这族方法在离散 episode 证据上的特例",**不声称实现了 BOCPD**。这样既保留了"变点检测"这个正确的族,又不开无法兑现的支票。

---

## 迭代 4|反"三招堆叠":一个残差,两轴读出全部动作

**弱点(立意层最致命):** 若方法章把 同化/顺应/例外/遗忘 并列成四条规则,就坐实了"trick pileup"的批评。

**改法:** 见"收敛结论"那张表。**只有一个量 r**;它被沿两轴读:幅度(高/低)× 复现(复现/孤立)。四个内容动作是这张 2×2 的格子,ACCUMULATE 是"高 r、复现性尚未判定"的过渡态。方法章开篇就摆这张表,把"我们只有一个判据"变成视觉事实。

---

## 迭代 5|例外不是死存储 → 查询时的一等证据

**弱点:** 若例外只是"存起来永不并入信念",审稿人会问"那它到底买了什么"。而且有个真实困境:一次真改变若只被提及一次,和一次例外在机制上不可识别(迭代 2),都会停在例外。

**改法:** 例外在**查询时**与信念一并渲染,带时间戳:"信念:上海;近期未确认:北京(2026-06)"。于是:
- 单次提及的真改变,以"近期未确认项"的形式浮现,**让读者/下游用 recency 打破机制无法打破的平局**——这是可识别性墙外能做的最好的事,且诚实。
- 例外从"死存储"变成"查询期证据",直接回答"它买了什么":它保住了**机制当下无法裁决、但可能要紧**的那条信息,不让它被平均掉或丢弃。

这条同时强化立意:例外不是仓库,是"第三种结局"在查询期兑现价值的地方。

## 迭代 6|finalize 回溯判 PROTECT 是时序不优雅 → 衰减支持使例外在线涌现

**弱点(在 `core.py`):** PROTECT 由 `finalize()` 流末扫描产生,不是在线决策。一条例外只有"回头看"才知道是例外——时序上不干净。

**改法(两个选项,标清取舍):**
- **A(更优雅):** 给假设支持加**跨 episode 衰减**:每经过若干 intervening episode 未被再次印证,支持衰减。孤立违反的支持会在够到 k 之前自然衰减到"例外"档——**同一个衰减统计量同时给出 顺应(涨得够快)vs 例外(衰减掉)**,不需要独立的 finalize 扫描。这正是带遗忘因子的 CUSUM。代价:引入一个时间常数(多一个参数)。
- **B(更简单):** 保留流末扫描,但在正文**正当化**为"改变-vs-例外的裁决本质上是延迟的:一个断言的身份只有在'后续是否印证'的窗口关闭后才确定"。

**建议:** 主实验用 B(可复现、参数少),把 A 作为"全在线版"消融点。诚实写清 B 的延迟语义不是 bug 而是问题本身的结构。

## 迭代 7|遗忘的位置:与例外在同一残差上对称

**弱点:** token 效率已被用户砍掉(不作卖点)。那 DISSOLVE 还留不留、怎么摆?

**改法:** 留作**理论对称性**,不作实验头条。最漂亮的部分恰在这:**同一个残差 r**,阈在低端 → 冗余可丢(DISSOLVE);阈在高端且孤立 → 不可归纳必须逐字留(PROTECT)。高残差留、低残差丢,是一根轴的两端。这把"遗忘"和"例外"统一成残差的镜像,极大增强"单一判据"叙事——即使不拿它做省 token 的实验。

---

## 迭代 8|候选归并噪声是机制的软肋 → 软支持 / 硬聚类的取舍轴

**弱点:** 整个数票机制依赖 LLM 把表层变体正确聚成同一假设("吃鱼/三文鱼/鱼素"→ fish)。归并错则票数错,顺应/例外全崩。且槽位命名本身也有同样问题(diet vs dietary_preference)。

**改法(标为设计轴,不假装无代价):**
- **硬聚类**(当前):讲述干净,但对归并错误脆弱。
- **软支持**:假设 v 的支持 = 语义相似度之和,而非计数。鲁棒,但把离散的"票数"变回连续模糊,叙事变复杂。
- 槽位命名同理,存在一层"槽位归并"。

主实验用硬聚类(好讲 + 可消融),把"归并错误率 → 端到端影响"作为一个必做的鲁棒性消融(审稿人必问)。

---

## 迭代 9|"图式的价值"讲成一个数:MDL 压缩量

**弱点(立意层):** 用户反复强调"图式记忆一定要有价值"。之前"价值=压缩/预测/稳定"是三种说法,偏定性。

**改法:** 用 MDL 把价值钉成一个量:**图式的价值 = 它省下的比特 = 用图式编码观测流相对逐字存储缩短的描述长度**。可预测的观测(r≈0)几乎免费,违反(高 r)要付残差编码。于是"价值"= Σ(逐字成本 − 图式编码成本),是可算的正数。这同时让"预测误差"落地为"残差编码长度",三种价值说法(压缩/预测/稳定)统一成"减少描述长度"的三个侧面。诚实标注:实现里用 LLM surprise **近似** L(obs|schema),MDL 是目标/透镜,不是声称精确算了码长。

---

## 迭代 10|稳定-可塑困境不是要藏,是要显式暴露成旋钮

**弱点:** 综述把 stability-plasticity(该更新 vs 该当噪声)列为公开难题。我们若默默设 k=2,等于偷偷选了个点。

**改法:** **显式**把 k(及可选的衰减时间常数)作为暴露该困境的旋钮:k 小 → 可塑(快改信念,易被例外带偏);k 大 → 稳定(抗噪,慢改)。方法章明说"我们不消除这个困境,我们把它参数化,并给出 k 的可识别性下界与 MDL 最优点"。把综述点名的难题变成我们方法的一个可扫描坐标——这是加分,不是回避。

---

## 迭代 11|数据结构最小性复核:一切皆断言列上的视图

**弱点复核:** 迭代 1 之后,确认没有多余状态。

**最终结构:**
```
Slot = 断言列 assertions: [(value, episode, r, t)]
派生视图(可缓存,非独立真相源):
  hypotheses[v] = {断言 v 的 episode}          # 分组
  belief        = 在任假设的指针
  superseded    = 历任在任假设序列
  exceptions    = 支持停在 1 且未在任的断言
  forgettable   = r≈0 的断言(图式可重建)
```
唯一的真相源是断言列;belief/superseded/exceptions/forgettable 全是它的视图。`won_lines`、独立的 `candidates` dict 都消失。这是能达到的最小状态。**好讲**:方法章只需定义"断言列 + 分组 + 在任指针",其余都是一句话的视图定义。

## 迭代 12|讲述性终检:三句话能讲完吗

**测试:** 一个方法优雅与否,看能否三句话讲清且不失真。

> 1. 图式为每个实体的每个属性维持一个**期望**;一条新观测相对期望的**残差 r** 是唯一信号。
> 2. r 低 → 观测被同化(并可作冗余遗忘);r 高 → 看它是否在**多个独立 episode**上复现:复现则图式错了、顺应改写信念(旧值进被取代链),孤立则不可归纳、作为**受保护例外**逐字留存。
> 3. "从不首次即改"不是保守而是**可识别性必需**(一条观测无法区分改变与例外);顺应的触发点由**描述长度最短**决定。

三句话,零黑话,数学锚点齐全(残差、可识别性、MDL)。**收敛。**

## 落到实现:相对当前 `core.py` 的具体改动清单

1. **删** `Slot.belief`(裸串)/ `won_lines` / 独立 `candidates` dict;**立** `Slot.assertions` 单一真相源 + 派生视图属性。(迭代 1、11)
2. **k 的语义**:文档/代码注释写成"可识别性下界=2,顺应阈= max(2, ⌈S/r̄⌉)";把 S 与 r̄ 作为显式可扫描参数。(迭代 2、10)
3. **PROTECT**:主版保留流末扫描 + 正当化延迟语义;加"衰减支持"作全在线消融开关。(迭代 6)
4. **DISSOLVE**:实现残差低端阈,作为与 PROTECT 对称的镜像;不进主实验声明。(迭代 7)
5. **查询渲染**:例外带时间戳与信念并列输出,作为查询期证据。(迭代 5)
6. **价值度量**:加一个 `schema.bits_saved()` 估计量(用 LLM surprise 近似残差码长),把"价值"变成可报的数。(迭代 9)
7. **鲁棒性消融**:归并错误率 → 端到端影响,列为必做。(迭代 8)

**未决(留给用户回来定)——只有两处是真岔路,其余都可直接做:**
- (a) PROTECT 主版走"流末扫描 B"还是"衰减在线 A"?我建议 B 主 / A 消融。
- (b) 候选支持用硬计数还是软相似度?我建议硬计数主 / 软作鲁棒性消融。

其余(数据结构统一、k 的可识别性+MDL 表述、价值度量、查询渲染、遗忘对称)我判断无争议,可直接进下一版 `core.py` 与方法章。

## 图检索选项分析(2026-07-21,与张老师讨论后定)

问题:既然已是实体图,PPR 或图式定制的图检索会不会有好处?

- **PPR(通用,HippoRAG 式)**:不做。它只对多跳关系推理有用(传播关联强度、不带时序,对 temporal 无用),且需要新增 ENTITY_REL / 槽位间关联边才能传播;而我们主打的 knowledge-update 是"单属性取当前值",召回一个槽位即够,多召回相关槽位反而引入噪声(已实测检索召回过多会淹没答案)。加 PPR 既不解瓶颈(瓶颈在抽取),又让定位滑向 HippoRAG。
- **图式定制的图检索(未来方向,截止后)**:比 PPR 更贴合立意——在现有 HAS_SLOT/EVIDENCE 边上做**类型加权传播**:信念边权重高、superseded 次之、例外边低但不归零(保住"待定反常可被 recency 打破平局")。这是 dual-trace 检索的图算法化,不需要 ENTITY_REL。属锦上添花:当前纯向量检索未成为瓶颈,故推迟。
- **时间线视图(现在做)**:temporal 唯一对路的解。同一份带时间戳的观测历史派生第二个视图(按 t 排序的全局事件线),补方法完整性,不碰仲裁核心。
